## Was ist dieses Projekt?

**Questolin** ist eine mobile-first Lernapp für IT-Grundlagen. Themen sind JSON-Decks mit Swipe-Slides und Quiz. Content ist erweiterbar ohne Rewrite (Registry-Pattern).

**PRD:** [docs/PRD.md](docs/PRD.md)

### Was dieses Repo **ist**

- Next.js App mit Content-Layer (`content/`, `lib/content/`, `lib/slides/`)
- Slide-Renderer-Registry (neuer Typ = neue Komponente + Registry-Eintrag)
- QA-Pipeline: `.qa/design/` → `.qa/acceptance/` → verify-ui

### Was dieses Repo **nicht** ist

- Vollständiges Gamification-Produkt (XP/Skill-Tree kommt später)
- HR-Lernsystem (Pattern-Inspiration von browo-hr, kein Fork)

## Architektur

```
questolin-app/
├── app/                    # Next.js routes
├── components/             # UI — alle Komponenten hier (Repo-Root)
│   └── slides/             # Ein Renderer pro Slide-Typ
├── content/
│   ├── schema/v1/          # JSON Schema + README
│   └── topics/de/*.json    # Lerninhalte
├── lib/
│   ├── content/            # Types, Zod, Provider, Loader
│   └── slides/             # Registry
├── scripts/validate-content.ts
└── .qa/                    # Design, Acceptance, project.yaml
```

### Schichtenregeln

1. **Content** (`content/`, `lib/content/`) — keine React-Imports
2. **Slides** (`components/slides/`) — nur Darstellung + lokale Quiz-State
3. **App routes** (`app/`) — Server Components laden Topics via `getContentProvider()`
4. Neues Thema = JSON-Datei; neuer Slide-Typ = `components/slides/` + `lib/slides/registry.ts`
5. Später: `SupabaseContentProvider` implementiert gleiches Interface wie `JsonContentProvider`

## Validation

- **Checks:** `npm run checks` (`validate:content` → lint → build)
- **Content:** `npm run validate:content`
- **Dev:** `npm run dev` → http://localhost:3000
- **Build:** `npm run build`
- **E2E:** `npm run test:e2e` (Playwright via @verify-ui when ready)

Run checks before push. Pre-push hook (Husky) and GitHub Actions enforce this. Do not use `git push --no-verify`.
